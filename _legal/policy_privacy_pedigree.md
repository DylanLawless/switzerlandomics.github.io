**Version 1.0**  
**Effective date: 14 June 2026**

This notice explains how information is handled when Pedigree is used.

It supplements the [general privacy policy]({% link _legal/policy_privacy_general.md %}) and [cookie policy]({% link _legal/policy_cookie_general.md %}).

## Local processing

Pedigree processes project information within the user's browser.

This may include family relationships, phenotype information, diagnoses, genomic findings, sample identifiers, notes, imported files and generated report content.

Switzerland Omics does not intentionally receive or store this project information on its servers through the Pedigree application.

## Local browser storage

Pedigree may automatically save a project snapshot in the browser's local storage so that work can be restored during a later visit.

Local storage may persist after the browser is closed. A person with access to the same device or browser profile may be able to access locally stored information.

Users can remove local information by clearing site data or local storage in the browser. Users should export any information they need before clearing it.

## User responsibility

Users are responsible for:

1. having appropriate authority and a lawful basis to enter personal, health, genetic or confidential information;
2. avoiding direct identifiers where they are not necessary;
3. securing the device and browser profile used for Pedigree;
4. controlling access to exported reports and project files;
5. retaining suitable backups; and
6. deleting information when it is no longer required.

Pedigree should not be used on a shared or public device for sensitive information unless suitable controls are in place.

## Imported and exported files

Files selected for import are processed in the browser by the Pedigree application.

Exported reports and project files are saved to a location chosen or controlled by the user. After export, the user is responsible for storage, access, transfer, retention and deletion.

## Website analytics

The surrounding Switzerland Omics website may use Google Analytics to measure general visits and page interactions.

Pedigree also uses an anonymous Cloudflare Worker counter to record the number of exported results. This counter increments when an export action is completed and is used only to display the public export count. It does not receive project information, imported file contents, generated report contents or user identifiers.

Pedigree project information, imported file contents and generated report contents are not intended to be included in analytics data.

Users should not place sensitive information in URLs or other website fields that could form part of general web requests.

## No account or cloud project storage

Pedigree does not currently provide a Switzerland Omics user account or cloud project storage through the application.

If cloud storage, accounts, collaboration or server-side processing are added in future, this notice will be updated before those features are introduced.

## Security

Local browser processing reduces transmission of project information but does not remove all risk.

Browser extensions, malware, shared devices, backups, exported files and other software may access information on a user's device. Users are responsible for choosing an appropriate environment for sensitive work.
